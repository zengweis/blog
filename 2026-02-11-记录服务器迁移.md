# 前言
之前腾讯云国际站搞活动，9.9usd买一年送3个月轻量就入手了新加坡的，现在国内腾讯云的服务器快到期了，迁移一下服务器。
顺便想优化一下博客的文章发布流程，之前是直接在服务器上写文章，然后hexo生成静态页面，现在打算用本地写好文章，然后push到github，再用github action自动部署到服务器上。

## 迁移过程
倒是没有出什么大问题，就是累人，迁移过程中需要重新配置服务器，安装软件，迁移数据等，花费不少时间。
先把宝塔安装好，备份wwwroot目录，然后把备份文件上传到新服务器，恢复数据。
在配置npm时候正常情况下，全局安装命令应该在 /usr/bin 这种地方。但宝塔为了管理多个 Node 版本，把它们隔离开放在了 /www/server/nodejs/ 目录下导致我全局安装的hexo包找不到，用了软连接解决这个问题：
```
ln -s $(npm config get prefix)/bin/hexo /usr/local/bin/hexo
```
然后就能美美地hexo new post了。

这次迁移还刚好能把弃置的项目也一并清理掉，省得浪费资源（因为新服务器内存少，只有2c2g）
